Accessibility

Semantics

  • The host uses role="toolbar" by default and sets aria-orientation from the orientation input.
  • Use ariaLabel or ariaLabelledBy when the toolbar needs a clear accessible name.
  • Use role="group" or [role]="null" when the surface is only visual grouping.
  • Projected controls keep their own native semantics, names, states, validation, and focus behavior.
  • The disabled state marks the toolbar with aria-disabled and applies inert to the projected subtree.

Keyboard support

KeyTargetBehavior
TabProjected interactive controlsMoves focus through controls in the normal page tab order.
Shift + TabProjected interactive controlsMoves focus backward through controls in the normal page tab order.
EnterProjected buttons or linksActivates the focused native control.
SpaceProjected buttons and checkbox-like controlsActivates the focused native control when that control supports Space.
Arrow keysProjected controlsHandled by the focused control, if it implements arrow-key behavior. Toolbar does not intercept them.
