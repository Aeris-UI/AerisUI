Import

One import provides the Glass directive and all of its inputs.

Features

Glass is a visual utility, not a new component variant. Place it on a supported Aeris surface or semantic container and keep the component's normal API, layout, and accessibility behavior.

Interfaces

Glass uses CSS values directly and has no supporting interfaces. The module array exports the complete directive API.

Design tokens

Set these tokens on :root, a theme scope, or an individual surface. Overlay components also expose component-specific backdrop tokens that fall back to --aeris-backdrop-blur.

Accessibility

  • Glass changes presentation only and adds no role, focus target, or interaction.
  • Text and controls retain their existing semantics and keyboard behavior.
  • Keep the translucent background opaque enough to preserve WCAG 2.2 AA contrast over every color or image that may appear behind it.
  • Avoid placing critical text over rapidly changing content because blur does not guarantee a uniform contrast ratio.
  • Disabling Glass removes both its translucent surface and backdrop filter.

Keyboard support

KeyBehavior
Any keyGlass handles no keys. The host component or projected interactive elements retain their documented keyboard behavior.
